Java TicTacToe game

This was done based on this YouTube video: https://www.youtube.com/watch?v=gQb3dE-y1S4

but most method implementations are original. This was done more so to get the basics of Java down, but I’m currently working on implementing a GUI for the game as well.

Full source code:

To be more Object Oriented, I decided to create a “GameBoard” to abstract a lot of the simpler methods that are called within a full game. Let’s take a brief glance at this class:

GameBoard class

		private char[][] board;
    private ArrayList<Integer> playerPositions = new ArrayList<>();
    private ArrayList<Integer> cpuPositions = new ArrayList<>();

The GameBoard class has three main instance variables, a 3-D array which is the board, declared as:

char[][] TicTacToe = {{' ', '|', ' ', '|', ' '},
                      {'-', '+', '-', '+', '-'},
                      {' ', '|', ' ', '|', ' '},
                      {'-', '+', '-', '+', '-'},
                      {' ', '|', ' ', '|', ' '}};

And two ArrayLists of integers, which save and keep track of the plays made by the players and the ‘cpu’.

The GameBoard class has five methods:

  1. printBoardState(): prints the current board to the terminal
  1. placeOnPosition(): takes an integer associated to a position on the board, and adds the appropriate piece, X or O, there with a switch expression
  1. checkGameStatus(): has all the possible win states for the game, and simply checks if the positions arrays match any of those and returns true if yes, false if not
  1. posPlayed(): checks if the current position being played (this should be an integer) has already been played, returns true if yes, false if not
  1. boardMaxed(): checks if the board is full and returns true if yes, false if not

The actual game

The whole game is structured as a while loop, with conditionals that break the loop based on checkGameStatus. The player has to simply type the integers to choose a position to place and that’s it.

The ‘CPU’ is actually just a random number generator for positions, but there are measures to prevent it from either repeating plays or being stuck forever.
